# 03 · User flows por audiencia

Sirven para validar que la navegación de los wireframes soporta estos recorridos. Conversión = solicitud (evaluación / diagnóstico).

Patrón general: **Entrada por sector → marca(s) que resuelven → cruce de ecosistema → conversión.**

## Salud
`Home o /sectores/salud` → **Pacífica** (lavandería/blancos) y/o **Trazatex** (trazabilidad clínica) → bloque Ecosistema → **Solicitar evaluación**.

## Hotelería
`/sectores/hoteleria` → **Pacífica** (blancos/lencería) y/o **Trazatex** (lencería con RFID) → Ecosistema → **Solicitar evaluación**.

## Industria
`/sectores/industria` → **Operissa** (uniformes/EPP, operación) y/o **Trazatex** (control por trabajador) → Ecosistema → **Solicitar diagnóstico**.

## Minería
`/sectores/mineria` → **Pacífica** y/o **Operissa** (operaciones remotas) → Ecosistema → **Solicitar evaluación**.

## Implicaciones para los wireframes
- Cada **página de sector** debe enlazar a las **marcas relevantes** (sección 8 de la Plantilla 4).
- Cada **Home de marca** y **detalle de soluciones** debe tener el **bloque Ecosistema** y un **CTA de conversión** siempre visible/accesible.
- El **CTA de Contacto** debe ser alcanzable desde el header en todo momento.
- Punto de conversión único: plantilla **Contacto** (formulario con campo Sector).
